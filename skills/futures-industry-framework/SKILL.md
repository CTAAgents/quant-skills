# 期货产业链+基本面+技术面一体化交易决策框架

## S_body：技能主体

### v2.0 核心模块（新增）

本框架已整合三大核心量化模块，解决主观性问题：

#### 1. 产业链定性标准化模块 (`industry_chain_standardization.py`)
- **功能**：通过量化指标体系替代主观判断
- **核心类**：`IndustryChainStandardAnalyzer`
- **主要功能**：
  - 库存周期四阶段自动判定（主动去库/被动去库/主动补库/被动补库）
  - 利润结构量化分析（上游/中游/下游/全链亏损）
  - 产业链综合评分计算（0-100分）
  - 标准化分析报告生成

#### 2. 评分权重客观化模块 (`weight_optimization.py`)
- **功能**：通过历史回测和机器学习确定最优权重
- **核心类**：`WeightOptimizer`, `BayesianWeightOptimizer`, `WeightSaver`
- **主要功能**：
  - 基于夏普比率的权重优化
  - 网格搜索权重探索
  - 贝叶斯权重优化（含不确定性估计）
  - 权重保存与加载

#### 3. 共振信号清晰化模块 (`resonance_detection.py`)
- **功能**：通过明确的量化规则消除模糊地带
- **核心类**：`ResonanceDetector`, `MultiTimeframeResonanceDetector`
- **主要功能**：
  - 三层共振等级判定（无/弱/中等/强/极强）
  - 信号方向判定（看多/看空/中性）
  - 多时间周期共振检测
  - 共振历史分析与可视化

#### 4. 集成分析引擎 (`integrated_analyzer.py`)
- **功能**：一站式整合使用所有模块
- **核心类**：`IntegratedAnalyzer`
- **主要功能**：
  - 一键执行三层完整分析
  - 自动生成标准化交易计划
  - 智能风险提示生成
  - 便捷函数 `quick_analyze()` 支持快速使用

### 核心概念

#### 核心问题
期货交易普遍存在以下缺陷（v2.0已解决）：
- 纯技术分析：缺乏产业逻辑支撑，容易被情绪左右
- 孤立看品种：忽略产业链联动关系
- 无数据验证：仅凭主观判断，缺乏量化支撑
- 无风控体系：裸奔交易，风险不可控
- **产业链分析主观**：依赖经验判断，缺乏标准化（v2.0已解决）
- **权重设置随意**：评分权重缺乏客观依据（v2.0已解决）
- **共振信号模糊**：信号判定缺乏明确规则（v2.0已解决）

#### 解决方案
采用「产业链定方向、基本面定强弱、技术面定时机、数据定真伪、风控定生死」的五层闭环逻辑，确保交易决策的严谨性和可追溯性。

#### 核心铁律
产业逻辑第一、数据验证第二、技术择时第三，三层共振开仓、逻辑背离观望。

### 三层一体化核心分析逻辑（固定研判顺序）

严格按照 产业链定性 → 基本面定量验证 → 技术面择时落地 顺序分析，不可倒置。

#### 第一层：产业链定性（定大趋势、多空主线、周期级别）
**核心目标**：判断品种中长期趋势（多/空/震荡/拐点），找出核心驱动与核心矛盾。

**必须分析内容**：
1. 全链利润分配：上游、中游、下游盈利/亏损状态，利润传导方向，极值利润风险
2. 库存周期定位：区分主动补库、被动补库、主动去库、被动去库，判断当前周期位置
3. 产业行为信号：开工检修、限产复产、企业套保、集中备货、移库、减产降负
4. 供需结构矛盾：供给瓶颈/过剩、需求强弱、淡旺季切换、政策扰动
5. 跨品种联动：同产业链品种强弱对比，确认主线一致性

#### 第二层：基本面定量（定强弱、定区间、证伪逻辑）
**核心目标**：用高频权威数据验证产业链逻辑真假，量化行情力度与价格区间。

**必须分析内容**：
1. 供需平衡表：期初库存、产量、进口、出口、消费、期末库存，重点看环比边际变化（优先级高于同比、绝对值）
2. 库存体系：交易所仓单、厂库、社会库存、港口库存、库存去化/累积速率
3. 基差与价差：期现基差、升贴水、跨期价差、跨品种价差，判断现货强弱与期货修复逻辑
4. 成本与估值：产业成本线、历史3/5/10年价格分位，判断高估/低估区间
5. 高频数据验证：开工率、产销数据、成交、到港、订单、终端需求反馈

#### 第三层：技术面择时（定入场、仓位、止损、止盈、离场）
**核心目标**：只做产业基本面共振的顺势交易，禁止逆势摸顶摸底，技术面仅作为执行工具。

**必须分析内容**：
1. 大周期定方向：周线/日线趋势、均线结构、高低点形态，与产业逻辑核对，背离则无条件观望
2. 中周期找节奏：4H/1H结构、震荡区间、趋势延续/拐点信号
3. 高胜率入场形态：顺势回踩、趋势突破、区间破位，结合量仓确认
4. 风控体系：关键支撑压力位、合理止损位置、仓位标准、移动止盈规则
5. 离场信号：产业逻辑失效、基本面边际反转、技术趋势结构破位

```python
def calculate_indicators_pandas(df, close_col='close'):
    """使用pandas计算全部技术指标（无Ta-Lib时使用）"""
    indicators = pd.DataFrame(index=df.index)
    close = df[close_col]
    high = df['high']
    low = df['low']
    
    # MA均线
    indicators['MA5'] = close.rolling(window=5).mean()
    indicators['MA10'] = close.rolling(window=10).mean()
    indicators['MA20'] = close.rolling(window=20).mean()
    indicators['MA60'] = close.rolling(window=60).mean()
    
    # RSI
    delta = close.diff()
    gain = delta.where(delta > 0, 0).rolling(window=14).mean()
    loss = (-delta.where(delta < 0, 0)).rolling(window=14).mean()
    rs = gain / loss
    indicators['RSI'] = 100 - (100 / (1 + rs))
    
    # STOCH
    low14 = low.rolling(window=14).min()
    high14 = high.rolling(window=14).max()
    indicators['STOCH_K'] = 100 * (close - low14) / (high14 - low14)
    indicators['STOCH_D'] = indicators['STOCH_K'].rolling(window=3).mean()
    
    # MACD
    ema12 = close.ewm(span=12, adjust=False).mean()
    ema26 = close.ewm(span=26, adjust=False).mean()
    indicators['MACD'] = ema12 - ema26
    indicators['MACD_SIGNAL'] = indicators['MACD'].ewm(span=9, adjust=False).mean()
    indicators['MACD_HIST'] = indicators['MACD'] - indicators['MACD_SIGNAL']
    
    # ATR
    high_low = high - low
    high_close = (high - close.shift()).abs()
    low_close = (low - close.shift()).abs()
    true_range = pd.concat([high_low, high_close, low_close], axis=1).max(axis=1)
    indicators['ATR'] = true_range.rolling(window=14).mean()
    
    # Williams %R
    indicators['WILLiams'] = -100 * (high14 - close) / (high14 - low14)
    
    # CCI
    typical_price = (high + low + close) / 3
    sma_tp = typical_price.rolling(window=14).mean()
    mad = typical_price.rolling(window=14).apply(lambda x: np.abs(x - x.mean()).mean())
    indicators['CCI'] = (typical_price - sma_tp) / (0.015 * mad)
    
    return indicators
```

### 标准化权威数据源规则

所有分析数据必须标注来源层级，严格遵守优先级：交易所官方数据 > 政府/协会数据 > 头部产业资讯 > 正规研报 > 市场情绪

#### 第二层级：分产业链专属产业高频数据源
- 黑色建材：Mysteel我的钢铁网（高炉开工、建材日度成交、钢厂盈利率、港口/厂库库存、原料到港、焦化产销）

### 风险分级与资金管理

#### 风险分级标准

| 风险等级 | 特征描述 | 建议操作 | 仓位限制 | 止损要求 |
|----------|----------|----------|----------|----------|
| **低风险** | 三层共振强烈、趋势明确、波动率适中 | 积极参与 | ≤30% | 1-2% |
| **中风险** | 两层共振、信号较明确、波动率一般 | 谨慎参与 | ≤20% | 2-3% |
| **高风险** | 信号不一致、趋势不明、波动率较大 | 小仓试错 | ≤10% | 3-5% |
| **极高风险** | 黑天鹅事件、流动性差、波动剧烈 | 不参与 | 0% | 立即离场 |

#### 风险评估要素

**1. 市场风险**
- 波动率：历史波动率、隐含波动率
- 流动性：持仓量、成交量、买卖价差
- 相关性：品种间相关性、与大盘相关性

**2. 基本面风险**
- 数据风险：数据质量、发布时间、修正风险
- 政策风险：产业政策、进出口政策、监管政策
- 事件风险：地缘政治、突发事件、天气灾害

**3. 技术面风险**
- 假突破：突破失败率、回踩确认
- 震荡洗盘：波动幅度、持续时间
- 流动性枯竭：极端行情下的流动性

**4. 操作风险**
- 仓位过重：超过风险承受能力
- 止损过宽/过窄：设置不合理
- 频繁交易：交易成本过高

#### 资金管理策略

**1. 总仓位管理**
```
总仓位原则：
- 单一品种：≤30%（低风险）、≤20%（中风险）、≤10%（高风险）
- 同产业链：≤40%（避免集中度风险）
- 总仓位：≤60%（保留足够现金应对机会）
- 杠杆控制：根据品种特性调整，不盲目高杠杆
```

**2. 分批建仓策略**
```
三层共振强烈：
- 首批：30-40%目标仓位
- 二批：突破确认后加20-30%
- 三批：趋势延续后加20-30%

信号一般时：
- 首批：20-30%目标仓位
- 二批：谨慎加仓或不加
- 三批：不加仓
```

**3. 止损设置方法**
```
技术止损：
- 趋势线止损：跌破/突破趋势线
- 支撑阻力止损：跌破关键支撑/突破阻力
- 均线止损：跌破/突破重要均线
- 波动止损：ATR倍数止损（1.5-2倍ATR）

资金止损：
- 单笔亏损：≤总资金的1-5%（根据风险等级）
- 单日亏损：≤总资金的5%
- 最大回撤：≤总资金的20%（触及则暂停交易）
```

**4. 止盈策略**
```
固定止盈：
- 第一目标：盈亏比1:1.5-2
- 第二目标：盈亏比1:3-4
- 第三目标：盈亏比1:5+

移动止盈：
- 趋势跟踪：移动止损跟随趋势
- 回撤止盈：从高点回撤一定幅度止盈
- 分批止盈：到达目标后逐步减仓
```

**5. 加仓与减仓规则**
```
加仓条件（必须同时满足）：
- 持仓已有浮盈
- 趋势延续确认
- 未达到总仓位上限
- 风险等级未上升

减仓条件（满足其一）：
- 到达止盈目标
- 趋势出现减弱信号
- 基本面出现边际变化
- 风险等级上升
```

#### 账户风险管理

**1. 每日/每周复盘**
- 记录每笔交易：理由、执行、结果
- 统计盈亏数据：胜率、盈亏比、最大回撤
- 分析错误原因：避免重复犯错
- 评估策略有效性：持续优化

**2. 风险控制阈值**
```
单日亏损：≥5% → 停止当日交易
连续亏损：≥3笔 → 暂停交易，复盘总结
单周亏损：≥10% → 降低仓位，谨慎交易
单月亏损：≥15% → 大幅降低仓位，只做高确定性机会
最大回撤：≥20% → 停止交易，全面复盘
```

**3. 压力测试**
- 极端行情：连续涨跌停、跳空缺口
- 流动性危机：无法平仓、滑点过大
- 黑天鹅事件：突发重大利空/利好
- 应对预案：提前制定应对措施

#### 交易复盘体系

**1. 每笔交易复盘（交易结束后立即进行）**

**标准复盘模板**：

### 核心设计原则

1. **产业链逻辑优先** —— 技术面仅为执行工具，不主导决策
2. **数据验证闭环** —— 所有结论必须有数据支撑，避免主观臆断
3. **三层共振开仓** —— 产业链+基本面+技术面三者共振才开仓
4. **边际变化优先** —— 环比变化优于同比变化，优于绝对数值
5. **逻辑背离观望** —— 任何一层背离都暂停交易，重新评估
6. **风控前置** —— 每笔交易必须带止损，不止损不交易
7. **资金管理** —— 仓位控制、分批建仓、严格止损止盈
8. **持续复盘** —— 每笔交易复盘，持续优化改进
9. **量化评分** —— 多维度量化打分，客观评估信号强度

### 量化评分体系

#### 1. 产业链评分模型

| 评估维度 | 评估要素 | 权重 | 评分标准 |
|----------|----------|------|----------|
| **利润结构** | 全链利润分配 | 20% | 上中下游均盈利=80-100分；利润传导顺畅=60-80分；利润分化=40-60分；全链亏损=0-40分 |
| **库存周期** | 周期定位 | 20% | 主动去库=0-30分；被动去库=30-60分；主动补库=60-80分；被动补库=80-100分 |
| **供需结构** | 供需平衡 | 20% | 严重供不应求=0-20分；供不应求=20-40分；供需平衡=40-60分；供过于求=60-80分；严重过剩=80-100分 |
| **产业行为** | 企业行为信号 | 15% | 集体减产=0-30分；检修降负=30-50分；正常生产=50-70分；积极增产=70-90分；疯狂扩产=90-100分 |
| **政策环境** | 政策影响 | 15% | 重大利好=0-20分；政策偏多=20-40分；政策中性=40-60分；政策偏空=60-80分；重大利空=80-100分 |
| **跨品联动** | 品种联动性 | 10% | 高度一致=0-30分；多数一致=30-50分；分化明显=50-70分；背离严重=70-100分 |

**产业链综合评分** = Σ(各维度得分 × 权重)

| 综合得分 | 信号 | 操作建议 |
|----------|------|----------|
| >75分 | 🔴 极强支撑 | 积极做多，顺势持仓 |
| 60-75分 | 🟢 强支撑 | 做多为主，仓位可偏重 |
| 45-60分 | ⚪ 中性 | 观望或轻仓试探 |
| 30-45分 | ⚠️ 偏弱 | 做空为主，轻仓试探 |
| <30分 | 🔴 极弱 | 积极做空，顺势持仓 |

#### 2. 基本面量化打分表

| 评估项目 | 评估要素 | 权重 | 评分标准 |
|----------|----------|------|----------|
| **供需平衡** | 库存去化/累积速率 | 25% | 加速去化=0-25分；缓慢去化=25-50分；持平=50分；缓慢累库=50-75分；加速累库=75-100分 |
| **基差结构** | 升贴水程度 | 20% | 深贴水=0-20分；贴水=20-40分；平水=40-60分；升水=60-80分；深升水=80-100分 |
| **成本估值** | 历史分位数 | 20% | <10%分位=0-20分；10-30%分位=20-40分；30-70%分位=40-60分；70-90%分位=60-80分；>90%分位=80-100分 |
| **数据质量** | 来源权威性 | 15% | 多源验证=0-15分；双源验证=15-30分；单源验证=30-50分；估算数据=50-100分 |
| **边际变化** | 环比变动方向 | 20% | 大幅改善=0-20分；小幅改善=20-40分；持平=40-60分；小幅恶化=60-80分；大幅恶化=80-100分 |

#### 3. 三层共振评分矩阵

| 共振情况 | 得分 | 信号强度 | 操作建议 |
|----------|------|----------|----------|
| 三层全强（均>60分） | 100分 | ★★★★★ | 重仓参与，积极顺势 |
| 两层强一层中 | 75-90分 | ★★★★☆ | 正常参与，顺势操作 |
| 两层强一层弱 | 60-75分 | ★★★☆☆ | 谨慎参与，轻仓试探 |
| 一层强两层中 | 50-60分 | ★★☆☆☆ | 观望为主 |
| 分散（无明显共振） | 30-50分 | ★☆☆☆☆ | 不参与或极轻仓 |
| 多空背离 | <30分 | ☆☆☆☆☆ | 不参与，观望 |

### 执行实操指导

#### 完整执行流程

```
1. 准备阶段
   ├─ 确定分析品种
   ├─ 归属产业链分类
   └─ 收集权威数据源

2. 分析阶段（严格按顺序）
   ├─ 第一层：产业链定性分析
   ├─ 第二层：基本面定量验证
   └─ 第三层：技术面择时分析

3. 决策阶段
   ├─ 共振检查：三层是否一致
   ├─ 风险评估：潜在风险分析
   └─ 交易计划：制定完整方案

4. 执行阶段
   ├─ 开仓：确认条件满足
   ├─ 持仓：跟踪信号变化
   └─ 离场：触发条件执行
```

#### 快速入门：30分钟分析流程

| 步骤 | 时间分配 | 关键动作 | 输出 |
|------|----------|----------|------|
| 产业链归属 | 5分钟 | 确定品种→列出上下游→确认核心驱动 | 产业链全景图 |
| 产业链定性 | 10分钟 | 利润分配→库存周期→产业行为→供需矛盾 | 大方向结论 |
| 基本面定量 | 8分钟 | 数据交叉验证→边际变化→基差估值 | 强弱区间判断 |
| 技术面择时 | 5分钟 | 大周期方向→中周期节奏→入场形态 | 执行条件 |
| 交易计划 | 2分钟 | 开仓方向→仓位→止损止盈→离场信号 | 完整交易计划 |

#### 实际应用示例：螺纹钢分析

1. **产业链归属**：黑色建材产业链（上游：铁矿、焦煤；下游：地产基建）
2. **产业链定性**：分析钢厂利润、高炉开工、社会库存、地产数据
3. **基本面定量**：验证环比库存变化、基差结构、钢厂盈亏线
4. **技术面择时**：周线趋势确认→日线寻找入场点→设置风控
5. **共振检查**：三者一致则开仓，否则观望

### 不同市场场景适配策略

#### 场景1：趋势行情

**特征**：方向明确、趋势延续、技术面与基本面共振

**策略**：
- 重仓顺势：利用趋势惯性
- 移动止盈：保护利润
- 分批加仓：确认突破后加仓
- 严格止损：防止趋势反转

**关键指标**：
- 产业链：利润传导顺畅、库存持续去化/累积
- 基本面：供需缺口扩大、基差持续走强/走弱
- 技术面：均线多头/空头排列、高低点不断抬升/降低

#### 场景2：震荡行情

**特征**：区间波动、缺乏明确方向、技术面与基本面时有背离

**策略**：
- 轻仓区间操作：高抛低吸
- 宽止损：防止被震荡洗出
- 减少交易频率：等待明确信号
- 网格交易：适合区间震荡

**关键指标**：
- 产业链：利润相对稳定、库存变化不大
- 基本面：供需相对平衡、基差波动收窄
- 技术面：价格在支撑阻力间波动、均线走平

#### 场景3：拐点行情

**特征**：原有趋势失效、新趋势尚未确立、信号混乱

**策略**：
- 观望为主：不急于抄底摸顶
- 小仓试错：确认信号后轻仓尝试
- 快速止损：方向错误立即离场
- 等待确认：多重信号共振后再进场

**关键指标**：
- 产业链：利润极值、库存周期转换
- 基本面：供需边际变化、政策转向
- 技术面：趋势线突破、背离信号、成交量放大

#### 场景4：黑天鹅事件

**特征**：突发重大事件、价格剧烈波动、流动性紧张

**策略**：
- 立即减仓/平仓：控制风险第一
- 停止开新仓：观望为主
- 关注流动性：避免在流动性差时操作
- 事后复盘：总结经验教训

**应对原则**：
- 风险控制优先于盈利
- 不猜测底部/顶部
- 保留资金等待机会

### 边界条件和异常处理

#### 1. 数据缺失处理

| 情况 | 处理方式 | 降级策略 |
|------|----------|----------|
| 产业链数据缺失 | 跳过该维度评分 | 仅使用其他维度 |
| 基本面数据缺失 | 标注"数据暂缺" | 使用估算数据并标注置信度 |
| 技术面数据缺失 | 降低技术面权重 | 主要依赖产业和基本面 |

#### 2. 场景切换机制

| 当前场景 | 切换条件 | 应对策略 |
|----------|----------|----------|
| 趋势 → 震荡 | ADX<20持续3日 | 转为高抛低吸策略 |
| 震荡 → 趋势 | ADX>25突破 | 转为趋势跟踪策略 |
| 趋势加强 | 突破前期高点/低点 | 顺势加仓 |
| 趋势衰竭 | 背离信号出现 | 逐步止盈 |

#### 3. 信号冲突处理

| 冲突类型 | 处理方式 | 决策原则 |
|----------|----------|----------|
| 产业链vs技术面背离 | 以产业链为准 | 技术面仅择时 |
| 基本面vs技术面背离 | 观望为主 | 等待信号一致 |
| 多空信号混乱 | 降低仓位 | 减少交易频率 |

#### 4. 黑天鹅应急处理

| 级别 | 触发条件 | 应急措施 |
|------|----------|----------|
| 黄色预警 | 单日涨跌幅>3% | 减仓50%，严格止损 |
| 橙色预警 | 连续涨跌停 | 停止开新仓 |
| 红色预警 | 流动性枯竭 | 全部平仓，保留资金 |

#### 5. 分析失败处理

| 失败类型 | 失败原因 | 解决建议 |
|----------|----------|----------|
| 数据获取失败 | 网络问题/接口故障 | 使用备用数据源 |
| 数据质量差 | 数据不完整/延迟 | 标注置信度，降低权重 |
| 分析逻辑矛盾 | 前提假设错误 | 重新审视前提条件 |
| 结论不确定 | 信号模糊 | 观望为主，不强行交易 |

#### 6. 执行异常处理

| 异常情况 | 处理方式 | 后续措施 |
|----------|----------|----------|
| 止损触发后反弹 | 记录为正常风控 | 分析是否需要优化止损 |
| 止盈后行情继续 | 记录为正常止盈 | 优化止盈策略 |
| 未能开仓（跳空） | 等待回调 | 评估是否值得追入 |
| 持仓浮亏超预期 | 提前止损 | 重新评估风险承受能力 |

### 【常见失误】执行失误警示

#### ❌ 失误1：孤立看单一品种
**后果**：忽略产业链联动，错过主线行情，误判趋势
**修正**：必须先归属产业链，分析全链利润、库存、供需

#### ❌ 失误2：纯技术分析，忽略产业逻辑
**后果**：被情绪左右，逆势交易，胜率极低
**修正**：先做产业链定性，技术面仅作为择时工具

#### ❌ 失误3：只看绝对值，忽略边际变化
**后果**：错过拐点，高位接盘，低位踏空
**修正**：优先看环比变化、库存去累速度、开工边际变动

#### ❌ 失误4：单一数据做决策
**后果**：数据噪音干扰，误判行情
**修正**：所有多空逻辑必须多数据源交叉验证

#### ❌ 失误5：逻辑背离仍强行交易
**后果**：逆势操作，大幅亏损
**修正**：产业链+基本面+技术面背离时，无条件观望

#### ❌ 失误6：无风控裸奔交易
**后果**：一次亏损抹平多次盈利
**修正**：每笔交易必须带止损，不止损不交易

#### ❌ 失误7：仓位过重
**后果**：账户波动过大，心理压力大
**修正**：严格遵守仓位管理规则，单笔风险≤总资金1-5%

#### ❌ 失误8：不复盘总结
**后果**：重复犯错，无法提升
**修正**：每笔交易立即复盘，每日/每周/每月定期复盘

#### ❌ 失误9：不遵守交易计划
**后果**：随意交易，纪律性差
**修正**：先制定计划，严格按计划执行

#### ❌ 失误10：过度交易
**后果**：交易成本高，容易疲劳交易
**修正**：减少交易频率，只做高确定性机会

> ⚠️ **警告**：以下约束必须严格遵守，否则可能导致严重亏损
